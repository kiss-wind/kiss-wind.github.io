Hexo博客搭建与bug修正，尽管艰辛，但是我也有自己的博客啦！

2020-02-28
本文总阅读量次

今天又是劳累的一天，我仍然被网速和bug打倒了。
因为舍友用github和hexo建了一个博客，并且发表了关于建博客方法的博文，这个是具体链接，有兴趣的话可以看一看。然后嘛我就有些好奇并蠢蠢欲动了，求知欲在作怪！我就对照着他博文里的方法开始建立自己的博客。
由于github一直没有中文插件，我也很少用它，就是不爱看满眼的英文（还好意思说自己是程序员呢）。但是为了满足我自己的好奇心，我顶着满脸的英语奥利给了。
首先安装好了node.js、hexo和git，我建好仓库，装好插件准备部署，结果发现老子怎么都部署不成功，不是这种错误就是那种错误，bug满天飞，我就太绝望了。后来发现以下bug：1.在补全配置文件_config.yml的时候所写的冒号后面没加空格，太粗心了；2.在repo后面那个地址的最后是自己仓库的地址再加一个.git，由于没加.git，URL就一直报错，error 400；3.也许是电脑抽风了，一直说我branch： master有问题，但倔强地抗争了几次后它又说没问题了（莫名其妙）。此时晚9:00
————————————————————————到这里就把我累的够呛了—————————————————————————
既然博客建好了，那想想怎么发文章吧？结果一查，恩？？？？？还必须得用代码写文章，奇怪的知识增加了.jpg 瞬间就感觉整个人不好了，下了一个hexo-admin插件，好歹能发布个空的文章吧，先看看效果，结果又卡在了deploy上，编辑好版面准备发布，deploy总是报错，说没找到命令，再次百度：CSDN告诉我原来必须在_config.yml文件末尾加上admin声明以及deploy命令，再一次好一个忙活，因为我不管怎么加上命令，创建与命令有关的bat文件，它一直找不到命令在哪。后来逐个错误百度，然后排查发现：1.配置文件编辑时命令大小写没写好（它居然不用驼峰命名法！！！要知道外国有一办公室的程序员因为不用驼峰命名法被同事挨个砍死。脸上露出了难以察觉的微笑）。2.命令文件内容书写错误：由于我还不太懂怎么写命令文件，就自己摸索，最后得出的正确写法是hexo g -d，意思是让它先生成，再部署，我把它分两行写，还加什么&&，都是错的，又捣鼓半天。终于把空的文章发布上去了。此时晚10:45。
——————————————————————————到这里我已半死————————————————————————————
好了，bug都解决了，我们看看效果吧，默认主题叫做landscape，我的妈，丑的要命，我决定开始我的换肤之旅！
行，查一查怎么换肤，继续开始踩坑，到hexo官网上找到了一个古风的皮肤，哇有点心动，就它了，送上链接。然后我百度如何换肤，首先要用git把那个人的仓库克隆过来，放在自己的博客目录中的themes文件夹下，然后装载到自己的仓库和博客网页上。
第一步开始踩坑，我又掉进了.git的坑里，git clone 后面写网址，我又忘了在网址后面加.git，行了，忙活半天，快找块豆腐撞死吧。后来好像还有一个bug，不过此文章是我完全装完以后才写的，因此我忘了是啥了，遇到什么困难还是问百度和CSDN更靠谱。
好了，皮肤下好了，hexo clean hexo g -d，一顿操作猛如虎，结果发现没反应？主页根本就没变！还是那个丑的要命的皮肤。我枯了啊。绝望的我再次运行了清理、生成、部署，奇迹出现了，它装上了，我长舒一口气。。。此时晚11:15.
——————————————————————————到这里我已濒死————————————————————————————
好的，主题装上了，可是头像啊文字啊是人家的文字，咱总不能用人家的头像和文字吧？于是我又开始探究如何改掉他的头像文件和文字，文字是在主题的_config文件里改，很容易找到，对照着网站上什么地方说的什么话换掉这句话就行了，头像也不难，在source>image里面（这里声明，可能不同的主题位置不一样，在主题那个文件夹里找一找就可以了），然后最简单粗暴的方法就是把自己的头像命名成和他的头像文件一样的名字，然后替换掉他的头像文件，因为我还不懂，我怕如果名字不一样的话会报错。把该改的地方都改完了以后，我就hexo装载部署，并且运行server里面的那个网址看了看效果，还算满意，再用真正的网址打开网页发现什么都没变，主题还是以前的主人的样子，改动的地方都未发生变化，我就又运行了一次装载部署，还是没用，前前后后运行了近十次，我的妈一点效果都没有。我慌了，百度查为什么hexo d之后远程网页没变化？用了他们说的方法还是没用，我太绝望了，我已经身心俱疲了啊！又忙活了一个小时，我翻找了主题的目录下的每一个文件，甚至js文件都用Dw翻了一遍，什么作用都没起。。。于是我尝试挪出一个边框的图像文件来测试到底是在装载过程中出了问题还是在配置文件里出了问题，结果挪出一个图像以后装载，居然好了！除了那个边框图像没显示（因为挪出来了显示不了），其他的全部变成了我想要的样子！我仿佛被当头敲了一棒子，把挪出来的图片放回去，再装载，就完全变成了我想要的样子。
所以为什么？？？？？？？？装载了十多次为什么一个小时以后才有效果？？？？？？也许只能拿网速来是问了。。。这个延迟足以把我弄死了。。。此时凌晨0:30
——————————————————————————到这里已经死掉了———————————————————————————
终于我完成了博客的初步搭建，但是我发现还有问题没有解决：在博客的最底部出现了一行小字：不蒜子告之阁下是第？个访客。我一沉，这行字哪里跑出来的，配置文件里面没有啊，而且翻找js文件的时候代码里面也没有啊？只能在控制台html里找到这行字，但是我没学过html或是php根本就不知道该怎么去掉它啊。。。虽然听起来有点文艺，但感觉好别扭啊！这个问题只能留到以后解决了。。。改天再研究，不然我就真的死了。
上文好像说过我不会用代码写文章发表，那么这篇文章有是哪来的呢？其实我下了一个应用，叫CEditor，它是GUI界面，可以把文章导出为.md格式的文件，导出以后把.md文件放进博客目录下的source_discarded里然后装载就可以了，如果装载不成，可以进admin看看有木有发布一下，是在没有的话肯定就到那该死的延迟在作怪了！！（快被延迟气死了）
嗯。现在是北京时间时间1:34，我写了一篇长长的感悟总结，太累了，一个程序员或者工程师或者架构师真的很不容易，我希望每个人都能快快乐乐健健康康地活着，不用太苦太累，但生活所迫，这苦不得不吃尽管我还是一个学生，提前感受感受加班的感觉哈哈哈哈哈，IT界的同行们，你们都是好样的！也请大家热爱身边的程序员。
疫情已经开始好转了，祖国还是很强大的，祝愿患者们早日康复，武汉加油，祖国加油，大家都加油！！
